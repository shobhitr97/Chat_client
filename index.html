OBJECTIVE OF THE PROJECT

To develop a back-end code for a chat client which allows the peers to chat on groups with 2 or more participants.All the peers can enter and exit the chat anytime and can share files over a Wi-Fi or a LAN network.This project taught us a lot of things about network sockets and overall networking concepts.

OUR IMPLEMENTATION

The network comprises of peers that are connected to each other to form a loop.Every member has a key that recognizes it .Every message has the key of the sender pinned to it.The message traverses the whole through subsequent passing and this stops when the message reaches the sender . There are two kinds of codes. One,for the peer that accepts new peers and is run at the beginning of the chat. The other one runs for all the other peers that simply request the former one for addition to the chat and simply chat. The former one can't exit the chat until everybody else leaves.

We worked on this project throughout our second semester and the following section shows how we made progress with time:

  • During the initial stage of our project we were given some reading material to introduce us to python ,github and network sockets.

  • After we finished our reading we were asked to make a simple server-client model in which the chat between two peers is facilitated by a server who passes each received message to every other client.

    This was later modified to incorporate more than 2 peers.

  • Then we switched from server-client to peer to peer model .In this network each peer was connected to every other peer and there was a tracker which kept track of the connected peers and catered to the needs of incoming peers.

  • A more enhanced and less complex version of this model had all the peers connected in a circle to form a network .Since they were all connected in a circle,each message traversed the whole circle by sequential passing.This improved the performance of the network.

  • Furthermore,we are trying to add capability to share files over the network.
